How It Works

From invoice to inbox — in seconds.

01

Upload Your Invoice Data

Drop a CSV or Excel file with your outstanding invoices. The agent validates every field using Pydantic — bad rows are rejected, not silently ignored.

02

Auto-Detect Overdue Records

The trigger engine calculates days overdue for every invoice and maps each record to its exact escalation stage — Stage 1 through 4, or Legal Flag for 30+ days.

03

LLM Writes the Perfect Email

Llama 3.1 70B via Groq generates a fully personalised, professional email — right tone, right urgency, all six required fields present and validated by Pydantic before send.

04

Send, Log & Escalate

Emails are sent via SMTP or dry-run logged to JSON. Every action is audit-trailed in Supabase. 30+ day accounts are flagged for legal review — no email sent automatically.

Escalation Engine

The right tone, every time.

Five stages. Each one calibrated. The agent knows exactly when to be warm, when to be firm, and when to hand it to legal — no manual decision required.

Stage 1 · 1–7 days
Warm & Friendly
Gentle reminder. Assumes the invoice was simply overlooked. Pay now link included.
Stage 2 · 8–14 days
Polite but Firm
Payment still outstanding. Requests a specific confirmation date from the client.
Stage 3 · 15–21 days
Formal & Serious
Credit terms at risk. 48-hour deadline to respond or make payment.
Stage 4 · 22–30 days
Stern & Urgent
Final notice. Pay within 24 hours or collections team is escalated.
Escalation · 30+ days
Legal Review
No email sent. Flagged in Supabase. Finance manager assigned for direct contact.

Under the Hood

Built on a free, production-grade stack.

Every component was chosen for zero operational cost and maximum production reliability. Groq's free tier gives 6,000 requests/day at ~500 tokens/second. Supabase persists audit data across every Render redeploy. GitHub Actions schedules the daily morning scan — completely free.

The architecture is designed to scale without code changes — swap any layer by updating a single environment variable.

Security

Finance data deserves respect.

✓  Prompt Injection

Input sanitisation strips injection patterns before data reaches the LLM.

✓  PII Protection

Client emails masked in all logs. Supabase row-level security enabled.

✓  No Key Leaks

All credentials via .env. Render env vars. Never committed to source.

✓  Hallucination Guard

Pydantic validates all 6 required fields are present. Email blocked if any missing.

✓  Safe by Default

DRY_RUN_MODE=true by default. No real emails ever sent without explicit opt-in.

✓  Trigger Auth

GitHub Actions uses a secret header. Unauthenticated /run-agent calls rejected.
